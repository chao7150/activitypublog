# Use Case Layer

このパッケージは、アプリケーション固有の**ビジネスシナリオ（手順）**を記述します。
ドメインモデルとゲートウェイ（インターフェース）を組み合わせ、特定のユーザー要求を実現します。

## 設計のポリシー

- **オーケストレーション**: 「リポジトリから取得し、ドメインサービスで計算し、結果を保存する」といった**処理の流れ**を記述する。
- **ドメイン知識の流出禁止**: 純粋な計算ロジックやビジネスルールは `domain/service` や `domain/model` に委ね、ここではそれらを呼び出すだけにとどめる。
- **実装の詳細への無知**: 具体的な DB の SQL や API の URL などは知らず、常に `gateway` のインターフェースを通じて操作する。

## ディレクトリ構成

機能単位（コンテキスト単位）でパッケージを分割する。
- `sync/`: 外部ソースからの同期に関するシナリオ。
- `digest/`: ダイジェストの生成や配信に関するシナリオ。
- `timeline/`: タイムラインの表示や取得に関するシナリオ。
